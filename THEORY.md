# Алгоритм моделирования распада произвольного разрыва
## Метод динамики частиц (Molecular Dynamics) — 2D ударная труба

---

## 1. ФИЗИЧЕСКАЯ ПОСТАНОВКА

### 1.1. Геометрия
- Прямоугольная труба: `[0, Lx] × [0, Ly]`
- Перегородка (мембрана) при `x = Lx / 2`
- Стенки — абсолютно жёсткие (упругое отражение)

### 1.2. Начальные условия (аналог задачи Сода)
| Параметр            | Левая часть (x < Lx/2) | Правая часть (x > Lx/2) |
|---------------------|-------------------------|--------------------------|
| Плотность ρ         | ρ_L (высокая)           | ρ_R (низкая)             |
| Макроскоп. скорость | u_L = 0                 | u_R = 0                  |
| Температура T       | T_L                     | T_R                      |

> **Давление** возникает естественно из двух вкладов:  
> — кинетический (тепловое движение): `p_kin = n·k_B·T`  
> — вириальный (межчастичные силы): `p_vir = ...`  
> Управляя ρ и T в каждой половине, задаём нужный перепад давления.

### 1.3. Ожидаемый результат распада разрыва (→ вправо по x)
```
  Волна              Контактный         Ударная
  разрежения         разрыв             волна
  ←─────             ──→                ──→
  ╱╱╱╱╱╱│            │                  │
─────────┼────────────┼──────────────────┼──────────
  (1)    │    (2)     │       (3)        │   (4)
         голова       хвост
```
Зоны: (1) невозмущённый левый газ, (2) веер разрежения, (3) область между КР и УВ, (4) невозмущённый правый газ.

## 2. МОДЕЛЬ ВЗАИМОДЕЙСТВИЯ ЧАСТИЦ

### 2.1. Потенциал взаимодействия — WCA (Weeks–Chandler–Andersen)

Чисто репульсивный, обрезанный и сдвинутый потенциал Леннард-Джонса:

```
            ⎧ 4ε [ (σ/r)^12 − (σ/r)^6 ] + ε,   если r < r_cut
U(r)   =   ⎨
            ⎩ 0,                                   если r ≥ r_cut
```

где **r_cut = 2^(1/6) · σ ≈ 1.1225·σ** — минимум потенциала ЛД.

### 2.2. Сила взаимодействия пары (i, j)

```
F(r) = −dU/dr = 24ε/r · [ 2(σ/r)^12 − (σ/r)^6 ],   r < r_cut
```

Вектор силы на частицу i от частицы j:

```
F_ij = F(r_ij) · (r_i − r_j) / r_ij
```

где `r_ij = |r_i − r_j|`.

### 2.3. Почему WCA?
- Чисто репульсивный → нет фазовых переходов газ–жидкость
- Гладкий (непрерывная сила) → стабильная интеграция
- Короткий радиус обрезки → KDTree-поиск очень эффективен
- При умеренных плотностях → газ ведёт себя почти как идеальный

## 3. ИНИЦИАЛИЗАЦИЯ

### 3.1. Размещение частиц

#### Шаг 3.1.1 — Определить числа частиц

```python
# Параметры задачи (в единицах σ, ε, m)
Lx, Ly = 100.0, 20.0          # размеры трубы
rho_L, rho_R = 0.4, 0.1       # числовые плотности (частиц/σ²)

# Количество частиц
N_L = int(rho_L * (Lx/2) * Ly)
N_R = int(rho_R * (Lx/2) * Ly)
N = N_L + N_R
```

#### Шаг 3.1.2 — Расстановка на искажённой решётке

Для каждой половины трубы:
1. Вычислить шаг регулярной решётки: `a = sqrt(1 / ρ)`
2. Разместить частицы в узлах квадратной решётки
3. Добавить малое случайное смещение (до 10% от `a`), чтобы убрать артефакты решётки

```python
def place_particles(x_min, x_max, y_min, y_max, rho):
    a = 1.0 / np.sqrt(rho)                  # шаг решётки
    xs = np.arange(x_min + a/2, x_max, a)
    ys = np.arange(y_min + a/2, y_max, a)
    xx, yy = np.meshgrid(xs, ys)
    positions = np.column_stack([xx.ravel(), yy.ravel()])
    # Малый шум
    positions += np.random.uniform(-0.1*a, 0.1*a, positions.shape)
    return positions
```

### 3.2. Задание начальных скоростей (распределение Максвелла)

Температура определяет хаотическое тепловое движение:

```
v_x ~ N(0, sqrt(k_B T / m)),    v_y ~ N(0, sqrt(k_B T / m))
```

В безразмерных единицах (m = 1, k_B = 1):

```python
def assign_velocities(N, T, u_macro=0.0):
    """
    N — число частиц
    T — температура (безразмерная)
    u_macro — макроскопическая скорость вдоль x
    """
    sigma_v = np.sqrt(T)       # k_B = 1, m = 1
    vx = np.random.normal(u_macro, sigma_v, N)
    vy = np.random.normal(0.0, sigma_v, N)
    # Вычтем среднюю скорость (убрать дрейф):
    vx -= vx.mean() - u_macro
    vy -= vy.mean()
    return np.column_stack([vx, vy])
```

> **Типичные значения для задачи Сода:**
> - T_L = 1.0, T_R = 0.8 → давление слева заметно больше
> - Или: одинаковая T, но ρ_L >> ρ_R → перепад давления за счёт плотности

### 3.3. Структуры данных

```python
pos = np.zeros((N, 2), dtype=np.float64)   # координаты (x, y)
vel = np.zeros((N, 2), dtype=np.float64)   # скорости (vx, vy)
acc = np.zeros((N, 2), dtype=np.float64)   # ускорения (ax, ay)
```

## 4. ПОИСК СОСЕДЕЙ — KDTree

### 4.1. Зачем KDTree?

Потенциал WCA обрезан при `r_cut ≈ 1.12σ`. Пары дальше r_cut не взаимодействуют.  
Полный перебор пар — O(N²). KDTree даёт O(N log N) для построения + O(k) на запрос.

### 4.2. Алгоритм поиска

```python
from scipy.spatial import cKDTree

def find_neighbors(pos, r_cut):
    tree = cKDTree(pos)
    # Все пары в пределах r_cut:
    pairs = tree.query_pairs(r=r_cut, output_type='ndarray')
    return pairs   # shape (M, 2) — массив индексов пар
```

> **Замечание:** `query_pairs` возвращает каждую пару ровно один раз (i < j).
> Это ключевое — вычисляем силу один раз и прибавляем ±F к обеим частицам (3-й закон Ньютона).

## 5. ВЫЧИСЛЕНИЕ СИЛ

### 5.1. Силы межчастичного взаимодействия

```python
def compute_forces(pos, pairs, sigma, epsilon, r_cut):
    """
    pos    : (N, 2) — координаты
    pairs  : (M, 2) — пары соседей
    Возвращает: acc (N, 2) — ускорения (при m=1 сила = ускорение)
    """
    acc = np.zeros_like(pos)

    if len(pairs) == 0:
        return acc

    # Векторы между частицами в парах
    dr = pos[pairs[:, 0]] - pos[pairs[:, 1]]       # (M, 2)
    r2 = np.sum(dr**2, axis=1)                       # (M,)

    # Отсечение (на всякий случай)
    mask = r2 < r_cut**2
    dr = dr[mask]
    r2 = r2[mask]
    valid_pairs = pairs[mask]

    # Предотвращение деления на 0
    r2 = np.maximum(r2, 1e-10)

    # Леннард-Джонсовские коэффициенты
    inv_r2 = sigma**2 / r2               # (σ/r)^2
    inv_r6 = inv_r2**3                    # (σ/r)^6
    inv_r12 = inv_r6**2                   # (σ/r)^12

    # Скалярная сила / r  (для удобства умножения на dr)
    # F(r)/r = 24ε/r² · [2(σ/r)^12 − (σ/r)^6]
    f_over_r = 24.0 * epsilon / r2 * (2.0 * inv_r12 - inv_r6)   # (M,)

    # Вектор силы = (F/r) · dr
    force_vec = f_over_r[:, np.newaxis] * dr          # (M, 2)

    # Третий закон Ньютона: F_ij = −F_ji
    # Используем np.add.at для атомарного накопления
    np.add.at(acc, valid_pairs[:, 0], +force_vec)
    np.add.at(acc, valid_pairs[:, 1], -force_vec)

    return acc    # при m = 1: acc = F/m = F
```

### 5.2. Оптимизация — векторизация

- Все операции — векторные NumPy, без циклов по парам.
- `np.add.at` — потокобезопасное накопление (медленнее `+=`, но корректно для повторяющихся индексов).
- Альтернатива: разреженная матрица `scipy.sparse` для суммирования.

## 6. ГРАНИЧНЫЕ УСЛОВИЯ — ЖЁСТКИЕ СТЕНКИ

### 6.1. Упругое отражение от стенок

После обновления координат проверяем выход за границы:

```python
def apply_walls(pos, vel, Lx, Ly):
    """Упругое отражение от 4 стенок трубы."""
    # Левая стенка (x = 0)
    mask = pos[:, 0] < 0
    pos[mask, 0] = -pos[mask, 0]
    vel[mask, 0] = -vel[mask, 0]

    # Правая стенка (x = Lx)
    mask = pos[:, 0] > Lx
    pos[mask, 0] = 2*Lx - pos[mask, 0]
    vel[mask, 0] = -vel[mask, 0]

    # Нижняя стенка (y = 0)
    mask = pos[:, 1] < 0
    pos[mask, 1] = -pos[mask, 1]
    vel[mask, 1] = -vel[mask, 1]

    # Верхняя стенка (y = Ly)
    mask = pos[:, 1] > Ly
    pos[mask, 1] = 2*Ly - pos[mask, 1]
    vel[mask, 1] = -vel[mask, 1]
```

## 7. ИНТЕГРИРОВАНИЕ — СХЕМА ВЕРЛЕ (Velocity Verlet)

### 7.1. Почему Velocity Verlet?

- Симплектический → сохраняет энергию на длительных расчётах
- Второй порядок точности по dt
- Прост в реализации

### 7.2. Алгоритм одного шага

```
ВХОД: pos(t), vel(t), acc(t)

1.  pos(t + dt) = pos(t) + vel(t)·dt + 0.5·acc(t)·dt²
2.  Применить граничные условия к pos(t + dt), vel(t)
3.  Построить KDTree по pos(t + dt)
4.  Найти пары соседей (r < r_cut)
5.  acc(t + dt) = compute_forces(pos(t + dt), pairs)
6.  vel(t + dt) = vel(t) + 0.5·[acc(t) + acc(t + dt)]·dt
7.  Применить граничные условия к vel(t + dt) [если частица отразилась]

ВЫХОД: pos(t + dt), vel(t + dt), acc(t + dt)
```

### 7.3. Выбор шага по времени dt

```python
# Критерий: частица не должна переместиться больше чем на долю σ за один шаг
# dt < σ / (v_max) · C_safety
# Для WCA типичное dt:
dt = 0.001  # в единицах σ·sqrt(m/ε)

# Адаптивный шаг (опционально):
v_max = np.max(np.linalg.norm(vel, axis=1))
dt = min(0.005, 0.05 * sigma / max(v_max, 1e-10))
```

## 8. ГЛАВНЫЙ ЦИКЛ МОДЕЛИРОВАНИЯ

```python
def simulate(N_steps, dt, pos, vel, sigma, epsilon, Lx, Ly):
    r_cut = 2**(1/6) * sigma
    
    # Начальные силы
    pairs = find_neighbors(pos, r_cut)
    acc = compute_forces(pos, pairs, sigma, epsilon, r_cut)
    
    # Массивы для записи (каждые save_every шагов)
    save_every = 100
    history = []
    
    for step in range(N_steps):
        # ===== Velocity Verlet =====
        
        # Шаг 1: обновить координаты
        pos += vel * dt + 0.5 * acc * dt**2
        
        # Шаг 2: граничные условия (стенки)
        apply_walls(pos, vel, Lx, Ly)
        
        # Шаг 3: полушаг скорости (со старым ускорением)
        vel += 0.5 * acc * dt
        
        # Шаг 4: новые силы
        pairs = find_neighbors(pos, r_cut)
        acc_new = compute_forces(pos, pairs, sigma, epsilon, r_cut)
        
        # Шаг 5: завершить обновление скорости
        vel += 0.5 * acc_new * dt
        
        acc = acc_new
        
        # Сохранение снимков
        if step % save_every == 0:
            history.append({
                'step': step,
                'time': step * dt,
                'pos': pos.copy(),
                'vel': vel.copy()
            })
            
    return history
```

## 9. ДИАГНОСТИКА И ВИЗУАЛИЗАЦИЯ

### 9.1. Макроскопические профили (осреднение по y и по бинам x)

Разбиваем трубу на `N_bins` полос по x. В каждой полосе:

```python
def compute_profiles(pos, vel, Lx, Ly, N_bins=100):
    dx_bin = Lx / N_bins
    bin_idx = np.clip((pos[:, 0] / dx_bin).astype(int), 0, N_bins - 1)
    
    x_centers = (np.arange(N_bins) + 0.5) * dx_bin
    
    density = np.zeros(N_bins)
    velocity_x = np.zeros(N_bins)
    temperature = np.zeros(N_bins)
    pressure = np.zeros(N_bins)
    
    for b in range(N_bins):
        mask = (bin_idx == b)
        count = np.sum(mask)
        
        if count == 0:
            continue
        
        area = dx_bin * Ly
        density[b] = count / area                        # числовая плотность n
        
        vx = vel[mask, 0]
        vy = vel[mask, 1]
        
        velocity_x[b] = np.mean(vx)                     # <v_x> — макро-скорость
        
        # Температура: T = m·<(v - <v>)²> / (d·k_B), d = 2 (2D)
        dvx = vx - velocity_x[b]
        dvy = vy - np.mean(vy)
        temperature[b] = 0.5 * np.mean(dvx**2 + dvy**2) # T = <δv²>/2 при k_B=m=1
        
        # Давление идеального газа: p = n·T (в 2D при k_B = 1)
        pressure[b] = density[b] * temperature[b]
        
    return x_centers, density, velocity_x, temperature, pressure
```

## 11. КОНТРОЛЬ КОРРЕКТНОСТИ

### 11.1. Сохранение энергии
```python
def total_energy(pos, vel, pairs, sigma, epsilon, r_cut):
    KE = 0.5 * np.sum(vel**2)         # кинетическая (m = 1)
    
    # Потенциальная
    dr = pos[pairs[:, 0]] - pos[pairs[:, 1]]
    r = np.linalg.norm(dr, axis=1)
    mask = r < r_cut
    r = r[mask]
    inv_r6 = (sigma / r)**6
    PE = np.sum(4*epsilon*(inv_r6**2 - inv_r6) + epsilon)
    
    return KE, PE, KE + PE
```

> Полная энергия должна сохраняться (колебания < 1%).
> Если растёт → уменьшить dt.

### 13.2. Списки Верле (Verlet neighbor lists)
- Строить KDTree не каждый шаг, а каждые 10–20 шагов
- Использовать расширенный радиус `r_list = r_cut + r_skin` (skin ≈ 0.3σ)
- Перестраивать, когда максимальное смещение частицы > r_skin/2

```python
# Критерий перестроения:
max_displacement = np.max(np.linalg.norm(pos - pos_at_last_build, axis=1))
if max_displacement > 0.5 * r_skin:
    rebuild_neighbor_list()
```

### 13.3. Cell lists (ячеечные списки) — альтернатива KDTree
- Разбить домен на ячейки размером ≥ r_cut
- Искать соседей только в 9 соседних ячейках (в 2D)
- O(N) по сложности вместо O(N log N)
- Легко реализуется на NumPy
